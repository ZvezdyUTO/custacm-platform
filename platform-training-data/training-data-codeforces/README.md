# training-data-codeforces

This module is the Codeforces vertical OJ slice for `platform-training-data`.

It owns Codeforces ODS ingest, ODS/DWD/DWM/DWS schema migrations, idempotent SQL task resources, internal Java read-side query services, local fixtures, and tests. It does not expose public DWD/DWM/DWS query HTTP APIs yet.

## Layer Rules

Dependency direction:

```text
web -> app -> domain
infra -> domain
config -> app/domain/infra/web
```

Rules:

- `domain` must not depend on `app`, `infra`, or `web`.
- `app` orchestrates use cases and should not build SQL.
- `infra` owns parsing, JDBC SQL, and database row mapping.
- `web` owns HTTP controllers and response DTOs only.
- `config` wires Spring beans and should not contain business logic.
- Codeforces warehouse time fields use UTC+8 (`Asia/Shanghai`) semantics.

## Directory Layout

```text
src/main/java/com/custacm/platform/trainingdata/codeforces/
  app/                 # application use cases; no HTTP or JDBC details
    service/           # ingest, DWD query, DWM query, and DWS query services
    result/            # application result records
  domain/              # domain boundary; no dependency on app/infra/web
    model/             # ODS/DWD/DWM/DWS records and read models
    value/             # reusable value objects
    criteria/          # query criteria records and default filter helpers
    parser/            # parser ports consumed by app services
    repo/              # repository/writer interfaces
  infra/               # infrastructure implementations
    parser/            # Codeforces payload parsing
    repo/              # JDBC repository/writer implementations and SQL mapping
  web/                 # HTTP layer
    controller/        # OJ-specific controllers
    response/          # HTTP response DTOs
  config/              # Spring Bean wiring

src/main/resources/
  db/migration/        # Flyway migrations owned by this OJ module
  fixtures/codeforces/ # stable local Codeforces test fixtures
  sql/ods/             # ODS upsert SQL
  sql/dwd/             # DWD set-based transform SQL
  sql/dwm/             # DWM set-based transform SQL
  sql/dws/             # DWS set-based transform SQL

src/test/java/com/custacm/platform/trainingdata/codeforces/
  app/service/         # application service tests
  domain/model/        # model validation tests
  domain/criteria/     # criteria pass-through and default-helper tests
  infra/parser/        # parser fixture tests
  infra/repo/          # JDBC repository and SQL task tests
  web/controller/      # controller behavior tests
```

`target/` is generated by Maven and is not part of the maintained source layout.

## Query Capability Index

| Layer | Entry service | Criteria | Repository | Table | Supported filters |
| --- | --- | --- | --- | --- | --- |
| DWD handle report | `CodeforcesSubmissionQueryService.listHandleSubmissions` | `CodeforcesHandleSubmissionCriteria` | `CodeforcesSubmissionRepository.findHandleSubmissions` | `dwd_codeforces__submission` | Requested handle, optional UTC+8 submitted time range, optional problem rating lower/upper bounds; bounded queries do not include unrated rows; app returns matching submission detail items. |
| DWD problem report | `CodeforcesSubmissionQueryService.listProblemSubmissions` | `CodeforcesProblemSubmissionCriteria` | `CodeforcesSubmissionRepository.findProblemSubmissions` | `dwd_codeforces__submission` | Requested problem key, optional UTC+8 submitted time range; app returns matching submission detail items. |
| DWM handle report | `CodeforcesFirstAcceptedProblemQueryService.summarizeHandleFirstAcceptedProblems` | `CodeforcesHandleFirstAcceptedProblemCriteria` | `CodeforcesFirstAcceptedProblemRepository.findHandleFirstAcceptedProblems` | `dwm_codeforces__handle_problem_first_accepted` | Requested handle, optional UTC+8 first-accepted time range, optional problem rating lower/upper bounds; bounded queries do not include unrated rows; app returns first-accepted problem details with problem metadata and first accepted UTC+8 time. |
| DWM problem report | `CodeforcesFirstAcceptedProblemQueryService.summarizeProblemFirstAcceptedHandles` | `CodeforcesProblemFirstAcceptedHandleCriteria` | `CodeforcesFirstAcceptedProblemRepository.findProblemFirstAcceptedHandles` | `dwm_codeforces__handle_problem_first_accepted` | Requested problem key, optional UTC+8 first-accepted time range; app returns accepted handle count and handle list with each handle's first accepted UTC+8 time. |
| DWS app report | `CodeforcesAcceptedSummaryQueryService.summarizeAcceptedProblems` | `CodeforcesAcceptedSummaryCriteria` | `CodeforcesAcceptedSummaryRepository.findDailyRatingAcceptedSummaries` | `dws_codeforces__handle_daily_rating_accepted_summary` | DB filters handle/date and can prune rows by rating columns inside the requested bounds; app-layer aggregation applies the same bounds and returns interval rating totals plus overall total. Unrated is included only when both rating bounds are absent. |

Null time/date bounds mean no lower or upper time/date bound. Null problem rating bounds mean all rated and unrated rows. Setting either `minProblemRating` or `maxProblemRating` filters to the requested inclusive rating interval and excludes unrated rows.
