## Known TODOs

- `platform-common/common-web` has no source files yet.
- Placeholder backend modules should stay out of the Maven reactor until their first runnable slice is implemented. `frontend` is now a runnable Node/Vite slice and remains outside the Maven reactor.
- Training-data currently implements Codeforces as a vertical OJ warehouse module with ODS, DWD, DWM, and DWS tables, SQL task resources, admin-triggered SQL task DAG refresh, disabled-by-default recent-lookback submission collection scheduling, process-local collection job polling for frontend batch collection, UTC+8 warehouse time semantics, public Java/HTTP DWD/DWM/DWS query support with an automatic-collection DWS summary list and backend-paginated newest-first DWD submission details, admin student-data purge, and a Codeforces-specific `studentIdentity` to handle binding table used by app-layer personal queries, admin HTTP collection, and cleanup. Persistent pipeline run state, cross-OJ handle accounts, and ADS remain future work.
- Auth has a lightweight `ErrorResponse` for current auth use cases. Cross-module unified error response and request tracing are future work documented in [../logging.md](../logging.md).
